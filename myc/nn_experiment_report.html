FC / CNN / Transformer 训练日志数据分析报告

本报告分析了 16 个带明确网络标签的实验目录，来源于 checkpoints-and-logs/local。核心指标来自 training_history_full.csv、 evaluation_metrics.csv 和可用的 monitor_summary.json。按你的新要求，所有图表与正文主结论统一只看 前 3000 iter，同时完全排除四个 transformer_1/2d_old 旧批次实验。

一页结论

报告重点是前 3000 iter 内的训练回报、评估回报、收敛稳定性和计算效率。跨家族对比时需要保留一个重要限定： FC、CNN、Transformer 的 YAML 并不只改了网络结构，同时也带有不同学习率、正则化和损失系数， 所以下面的“跨家族结论”本质上是“架构 + 该家族配置组合”的结论；只有 CNN1-5 和 Transformer 1D/2D 这两组消融内部对比更接近单变量。

实验范围与方法

纳入范围

  • 纳入所有目录名带 _fc、_cnn*、_transformer* 的实验。
  • 排除 1 个无架构后缀目录：A2C 2026-04-18 21-56-01。
  • 按你的要求额外排除 4 个旧版 Transformer 实验：A2C/PPO + transformer_1d_old/transformer_2d_old。
  • 训练过程使用 training_history_full.csv；评估曲线使用 evaluation_metrics.csv。
  • 梯度稳定性优先用 monitor_summary.json，缺失时退回到训练历史。
  • 所有图表、矩阵表和正文主结论统一只使用 epoch <= 3000 的数据窗口。

指标定义

  • Best / Final evaluation reward： 都限定在前 3000 iter 内。
  • 1-peg epoch： 第一次达到 pegs_left <= 1 且 epoch 不超过 3000。
  • Regression gap： best_eval - final_eval，两者都按窗口口径计算。
  • Final critic loss： iter 3000 前最后一个训练点的 critic_loss。
  • Final entropy： iter 3000 前最后一个训练点的策略熵。
  • Final gradient： iter 3000 前最后一个可用梯度监控值。

网络结构侧写

这里按当前代码与配置文件总结三类网络的结构变化轴。FC 与 Transformer 主要是家族级差异，CNN 和 Transformer 位置编码内部则属于更标准的消融。

FC

全局展平后直接送入多层 MLP。优点是实现简单、单步训练开销低；缺点是完全丢掉 2D 局部结构先验。

CNN

共享卷积骨干加残差块，再接 policy / value head。cnn1-cnn5 这组实验本质上在测试核大小、归一化方式和残差增强模块。

Transformer

将棋盘展平成 49 个 token，先做线性嵌入，再注入 1D 或 2D 位置编码。本报告仅保留新批次的 1D/2D 对比，不再展示旧批次结果。

FC 家族

如果只看前 3000 iter，FC 还没有把 A2C 和 PPO 拉开到“是否 1-peg solved”这么大的差距。 A2C · FC 的窗口末评估奖励为 1.968， PPO · FC 为 0.968。 也就是说，FC 的优势在 3000 iter 窗口内已经能看见，但它仍然更像一条稳步爬升曲线，而不是最激进的早期解法。

从早期曲线看，FC 更像是一个稳步爬升但启动不算快的基线。

CNN 消融

主要发现

  • A2C-CNN： 如果采用上面的 A→E 推断映射，那么 3x3 系列 (cnn2/cnn4/cnn5) 在前 3000 iter 的稳定性明显好于 5x5 系列 (cnn1/cnn3)。前者大多维持在 2 pegs 左右，后者更容易提前回退。
  • PPO-CNN： cnn1 和 cnn3 都能在前 3000 iter 内进入 1-peg 区间， 说明 5x5 核在 PPO 下并不吃亏；但 cnn5 在加入 PreAct + SE 后反而从 0.903 回退到 0.806， 这意味着“更复杂的残差增强”至少没有改善前 3000 iter 的早期效率。
  • 更稳的结论： 这组 CNN 内部消融比跨家族对比更可信，因为它们共享同一个卷积家族框架，只改核大小、归一化和残差样式。

Transformer 位置编码与稳定性

在 A2C 下，Transformer 仍然是最容易出现训练不稳的一类。A2C · Transformer 1D 在 iter 3000 时的梯度范数为 115.278， 熵为 0.012； 而 A2C · Transformer 2D 分别是 0.895 和 0.064。 这说明在保留实验里，2D 位置编码至少提升了前 3000 iter 的稳定性。

但这个优势并没有在 PPO 下转化成绝对性能突破。PPO · Transformer 1D 和 PPO · Transformer 2D 在前 3000 iter 内都没有达到 1 peg， 且窗口末评估奖励分别只有 0.806 和 0.742。

由于旧版四个实验已被移除，这里只保留新批次 1D/2D 的同窗比较。

Critic Loss 与策略熵

这两个指标都取自前 3000 iter 窗口内最后一个训练点。熵更接近“策略是否过早变得确定”，critic loss 更接近“价值函数当前拟合误差”； 它们都能辅助解释稳定性，但都不能单独替代评估回报。

读图要点

  • 最低熵的一组基本都对应塌缩： A2C · CNN1、A2C · CNN3、A2C · Transformer 1D 的窗口末熵分别只有 0.000、 0.006 和 0.012， 其中前两个实验的窗口末评估奖励已经退到 0.065 和 0.000。
  • 熵也不是越高越好： PPO · CNN5 的窗口末熵最高，为 1.281， 但窗口末评估奖励只有 0.806。 相比之下，真正跑到 1-peg 的 PPO · CNN1 和 PPO · CNN3 分别停在 0.419 和 0.569， 说明中等熵水平更像“既保留探索、又没有完全塌缩”的平衡点。
  • critic loss 与最终表现不是单调关系： 最高的窗口末 critic loss 出现在 A2C · Transformer 1D，达到 0.152， 这和它的梯度尖峰、评估回退是一致的；但最低的三项里也包含 A2C · CNN3 和 A2C · CNN1， 它们的 critic loss 虽低，窗口末评估奖励却只有 0.000 和 0.065。
  • 更健康的组合通常出现在“中等 critic loss + 非零熵”： 例如 A2C · FC 的窗口末 critic loss / entropy 为 0.038 / 0.211， 而 PPO · CNN1 和 PPO · CNN3 也都在非零熵下保持了零回退。

效率与成本

A2C-FC 在前 3000 iter 内平均每个 epoch 只需 0.406 秒； PPO-Transformer 1D 约为 3.348 秒。 这解释了为什么 PPO-Transformer 即使曲线不占优，仍会消耗明显更多的优化时间。

建议

  • 如果下一轮目标是“前 3000 iter 内尽快拿到高评估表现”，优先保留 PPO + CNN1/CNN3。
  • 如果继续做 CNN 消融，建议先围绕核大小和归一化做更细粒度验证，再决定是否保留 PreAct + SE；当前证据并不支持它改善早期效率。
  • 如果继续做 Transformer，先把训练稳定性作为第一优先级，尤其是 A2C 下的梯度异常和熵塌缩。
  • 如果要做严格的“只改网络结构”结论，建议把 FC / CNN / Transformer 的学习率、熵系数和损失系数统一后再复现实验。
